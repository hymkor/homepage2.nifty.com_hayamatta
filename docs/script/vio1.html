[1] S2NYAOS なら入力が多い日も安心

さて、貴方の手元にはいきなりだが、何も無い。強いて言えば、Netscape と UNZIP くらいだ。この先、まず何をすべきであろうか？emx を導入する？ ノンノン。貴方はファイル名の入力さえ一苦労な CMD.EXE の上で、長い導入作業をやるつもりかね。これからのインストール作業でそれは余りに辛く長い道のりだ。キーボードのタイプが好きで仕方ない人も多いと思うが、それはあくまでマウスとの比較論的な話だ。ここで、賢い VIO ユーザならば

S2NYAOS.EXE を \OS2\APPS へ
コピーする以外考えられない

のである。S2NYAOS.EXE とは何か？ それは、emx が導入されていなような苛酷(笑)な環境でも動作できるよう、ライブラリをスタティックリンクされた NYAOS のことである。つーても分かる人しか分からんか？

とりあえず、http://hp.vector.co.jp/authors/VA009797/warp/nyaos.htmlに転がっているので、s2nya1??.zipと書かれている方をダウンロードしよう。nyaos1??.zip と書かれている方でもいいんだが、こっちは emx関係のDLL をいろいろと必要とするので、今の時点では役に立たない。ドキュメントがついていないが、まぁ、無くても何とかなるし、ここでも説明してやる。なお、UNZIP さえ、まだ手元に無いという方の為に、自己展開型書庫版も用意されている(s2nya1??.exeという奴だ)。

ネットスケープでダウンロードが終わったら、導入あるのみである。OS/2 プロンプトで「unzip s2nya164」を実行すると、s2nyaos.exe だけが展開される(自己展開書庫版の場合は「s2nya164」だけでOk!)。これを PATH の通ったディレクトリにコピーするのだ。「PATH の通ったディレクトリ」が分からん人は、素直に \OS2\APPS に置いてしまえ。本当は \OS2\APPS はシステムの為のディレクトリだが、な〜に、

後で消しときゃ、
誰も文句は言わねぇ

別に消さなくても、誰も文句は言わねぇが。とりあえず、emx をちゃんとインストールするまでは、こいつが最も多く使うソフトのはずだ。とりあえずは、デスクトップにアイコンを作っておくことをお薦めする。「デスクトップ」→「OS/2 システム」→「テンプレート」から「プログラム」というアイコンをデスクトップに落とせばいいわけだ。こんなこと OS/2 ユーザなら誰でも分かる？いや、俺は忘れていたぜ、VIO 使いだから(爆笑)。

冗談はともかく、アイコンに書く内容は次の通りでいいはずである。

パスおよびファイル名 x:\OS2\APPS\S2NYAOS.EXE
(x:は OS/2 をインストールしたドライブ)
パラメーター 空欄でよろし
作業ディレクトリー 貴方の作業ディレクトリにするある

タイトル名は任せる。当分の間は「OS/2 プロンプト」ではなく、このS2NYAOS のアイコンを叩いた後のVIOウインドウ内で作業することを前提とする。

さて、有無を言わせず、強引にインストールさせてしまったのには訳がある。これから行う emx の導入に際して、nyaos の補完機能があるのと、無いのとでは作業効率が大違いであるからである。まず、nyaos といえば「ファイル名補完」であるというくらい、補完機能は VIO での作業で欠かせない機能であるのだな。おかげで、

というか、補完以外の nyaos の機能
知ってる人いるのか？

と作者は危惧を抱いているくらいだ。さて、その補完機能。tcsh を知っている人は御存知だと思うが、一応、説明せねばなるまい。

要はファイル名の入力を一文字でもサボるための機能なのだ。例えば、先程の「unzip s2nya164」という入力だが、普通だと15キー分、地道にタイプしなくてはいけない。それが当り前だと考えている方は、これから一生苦労するに違いない。これが、nyaos の上ならば、ほとんどの場合「unz[TAB]s2[TAB]」 と約半分のタイプ数で済んでしまうのだ。今[TAB]と書いたが、これは 5文字分ではなくて、Ｑキーの左にあるタブキーを一回叩くことを意味する。補完機能つーのは、タブキーを押すと、その時点でカーソルの左にある単語をファイル名の一部とみなし、残りの文字を可能な限り、継ぎ足してくれる機能なのである。

どうだ！便利だろう！

候補が複数ある時はどうするかって？ ぬかりは無い。可能性のあるファイル名を画面に表示した後、ファイル名全部とはいかないが、可能な分だけやってくれるのである。例えば 

	s2nya164.zip  s2nya160.zip

という二つのファイルがあった時、「unzip s[TAB]」とタイプすると、どうなるかというと、まず「unzip s2nya16」まで補完してくれる。で、ここで更に[TAB]をタイプすると、上のリストを表示してくれるのである。あとは「0[TAB]」あるいは「4[TAB]」のどちらかをタイプすれば、残りの文字も入力できる次第である。らくちん、この上ないね。

もちろん、補完機能は nyaos が元祖ではなく、遠い古代の昔から、tcsh や bash などで実現されてきた機能である。もちろん、OS/2 版の tcsh でも利用できるのだが、いかんせん、UNIX と OS/2 のファイルシステムの差か！大文字と小文字を区別してくれるのである。入力の際、正確に大文字・小文字をあわせてやらなくてはいけないのだが、nyaos の場合は、そこのところ、自動的に既にタイプしたファイルの大文字・小文字まで補正してくれるのである。

いたせりつくせりだな！

作った当人が言ってりゃ世話ないのだが、とりあえず、自慢してでも、メリットをアピールせねば、使ってもらえないのが世の非情さなのだ(何か違う)。だから言わなきゃね。なお、最近の bash は、大文字・小文字を区別しない補完を装備したそうだが、nyaosの作者としては当然無視するしかあるまい(笑)

[格言]
ファイル名打つときゃ、とにかくタブたたけ！
指が折れるまで(爆)

これで貴方も、新手の nyaos 使いだ。さぁ、ゴゴゴといわせよう。
