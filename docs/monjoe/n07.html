erl の例の謎について、会社にたまたま落ちていた*31 駱駝本で一応納得しました…というか騙されたような気もしますが(藁。

によると、まずはシンボリックリファレンス*32 という機能で「${"foo"}」とか「$bar="foo" ; ${ $bar } 」という形で$foo という変数にアクセスできるとのことだそうで。

で、何故か逆も真なりということで、$foo{ bar } も$foo{'bar'}としてもよいらしいんですね。仕組みとしては「スカラー変数参照のための中括弧内ではクォートを省略できる」ということらしいですが、なーんか強引な理屈ですねぇ。字句/構文解析上そうなっちゃったのが結果オーライだっただけのような気もしますが。このご都合主義が Perl の味なんでしょうねぇ。

織華さんにご指摘いただいた例は JavaScript ですが、矢印（=>）がコロン（:）になっただけで、ほぼ同じのようですね。こういうのは動的に文脈を解釈できるスクリプト言語の共通の特徴なのかも。

でも、Ruby は…多分、nil がどうのこうのというエラーではじかれそうな気がします。
