サムライプログラマー 　(2003.03.08)

♪
そうだ、思い出すんだ、コーディングの喜び！

俺にコーディングさせろー

まぁ、そういう魂の叫びはおいといて、休憩の一環と称して、仕事で使わせる基盤ライブラリを作っています。

そいつの一つにエラーログ関数があるのですが、ちょっとでも手軽に使えるように、引数少な目にできないかなーと調べていると、C99では色々と便利なマクロが追加されているようです。

  • __VA_ARGS__
    可変長引数をとる関数マクロの仮引数。
    #define eprintf(...) fprintf(stderr,__VA_ARGS__)
    のように使う。
  • __FUNCTION__
    関数名が入る。

さて、各コンパイラのサポート状況はと…

コンパイラ __VA__ARGS__ __FUNCTION__
gcc 2.95 ○ ○
Borland C++ 5.5.1 × ×
Digitalmars C++ 8.26n ○ ○
OpenWatcom C 1.0 ○ ○

Digitalmars/OpenWatcom イイ（・∇・）

やはり、俺様が見こんだことだけはあるコンパイラだぜ、ククゥ。これで自宅で開発できるつーもんだ(ぉぃ) *7。で、これらを使って何を作ろうかとしているかというと、エラーログ関数/マクロを作ろうとしているのです。いるのです、ええ。


    void errlog_( const char *funcname , int level , const char *fmt,...);
    #define errlog(...) errlog_(__FUNCTION__,__VA_ARGS__)

やりたいこと分かるでしょ？さて、本当に使うかどうか…それが問題だ。
