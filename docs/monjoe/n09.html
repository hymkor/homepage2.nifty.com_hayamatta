神聖再帰句探偵団 　(2003.09.17)

某
メーカーのベテランエンジニアの人が、ファイルのバックアップを取るのに、

    cp -p foo/bar{,-20030915}
とタイプしていた。

ご存知の方はご存知だと思うが、bash はこれを

    cp -p foo/bar foo/bar-20030915
と展開してくれるのだ。これを手入力すると 2 回目と 1 回目で違うファイル名を補完してしまうという罠が待ち受けている(そんなドジ踏むのは、おまいだけ)。

これ(中括弧)はよいテクニックだと思いきや、NYACUS にはなかった。ということで、さっそく盛り込みました。当然ながら、やっちゃいましたよ。無限展開を(笑)

「A{1,2}{3,4}」は「A13B A14B A23B A24B」って展開しなくちゃいけないんだけど、途中経過を見ると「A1{3,4} A2{3,4}」→「A13 A2{3,4} A14 A2{3,4}」…

中括弧の数、全然減ってないじゃん!!

再帰でよくやる失敗をやってしまいましたわ、はは。展開関数を f とすると、

      f('A{1,2}{3,4}')
    = f('A1{3,4} A2{3,4}')

とやっちゃいました。こりゃ、確かに永遠に終わりませんわ。そうではなく

      f('A{1,2}{3,4}')
    = f('A1{3,4}') + ' ' + f('A2{3,4}')

と変換せねばならぬわけですな。いや、はは。

なお、パッケージやドキュメントまで作る時間がないので、公開はまだ先です。

腐ったプログラムサイトの方程式 　(2003.09.14)

私
は外出するとか、バックアップの時とかのために、ファイルサーバ上の特定のディレクトリを丸ごと、ノートPC の中にコピーしておくということをよくやってしまう。その為、ディスクの中は、いろんな時期のスナップショットがたくさん存在する。

古いのを(CD-R に焼いてから)容赦なく削除するというのも手だと思うが、もし、古い方のツリーの中のファイルを修正していたりすると、そんなファイルが埋もれたまま、失ってしまう。。。

さらに、私は最近、PC を新調した。だいたいのデータはニューマシンに移動済みだが、抜けがあるかもしれない。

そんな場合のために、ディレクトリツリーのマージツールを Ruby で作ってみた「mergetre.rb dir1 dir2 > f.sh」とやると dir1 の中を dir2 の中に移動させるスクリプト(f.sh)を作成する。(公開はするけど、ファイルが失われた時の責任はとれないので、こいつで作ったシェルスクリプトは決して実行させないように!)。この f.sh は

  • 同名のファイルが存在していて、サイズ・タイムスタンプが同一の場合はdir1 の中のファイルを削除するのみ
  • 同名のファイルが存在していて、サイズ・タイムスタンプのいずれかが違う場合はdir1 の中のファイルに @YYYYMMDDhhmmss と日付をつけて、dir2 の中へ移動させる。
  • dir1 にしか存在しないファイルは、名前をかえずに、そのまま dir2 に移動させる。
  • ディレクトリまでは削除してくれない不親切設計。

というようなことを行う。

mergetre.rb がファイルの移動・削除を直接実行しないのは、実行しないで結果を見たいためである。重大な結果を伴うようなスクリプトの場合、こういう中間物を一度作るようにしておけば、お客さんが同業者の場合には確認してもらって、責任を分散させることができるというメリットがある。

で、ある程度やってみて、特に問題なく動くようなので、実際に使ってみた。ふむ、それなりに使えるが…うーむ、どーも、Windows でやってみると、ファイル名に逆クォートとかシングルクォートが含まれていて、bash だと誤削除こそありませんが、移動ができないようだ。つーことで、そういうファイルに限っては手作業で移動させた。

⊂⌒~⊃。Д。）⊃

巷
に Ctrlキーと CapsLock キーを入れ替えるツールは結構ある。

一方、たくさんではないが、窓使いの憂鬱のように、UNIX 的なキー操作を Windows のそれに変換してくれるツールがある。しかしながら、これだと Windows の機能であるところのCtrl-X (カット) と、Emacs 的な２ストロークキーであるところの Ctrl-X が重複してしまい、何かと不都合がありそうだ。

ここは、やはり CapsLock を機械的に Ctrlキーにしてしまうのではなく、CapsLock + 他のキーを、これこれに変換するといったように個別に置換できた方がよろしい。そうすれば、Ctrl-X と CapsLock-X という形で二つの機能が混在できる。いうなれば、CapsLock が「真Ctrlキー」となるわけだ。

そんなことを実現するツールはないものか…。つーか、Windows を使わなければ皆幸せになれるわけだが。
